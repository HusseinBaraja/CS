# Project Roadmap Baseline

This roadmap was re-baselined after implementation had already started.

- Phase 1 through Phase 3.4 mostly describe work that has already landed in the repository.
- Phase 3.5 onward mixes completed foundation with remaining work and explicitly calls out the current baseline in each step.
- The authoritative product intent lives in `docs/PROJECT_CHARTER_AND_VISION.md` and `docs/SRS.md`.
- The roadmap follows the current monorepo boundaries in `apps/*`, `packages/*`, and `convex`, not the original pre-build folder assumptions.

## How To Read The Remaining Steps

Every roadmap step from Phase 3.5 onward uses the same structure:

1. Step title and goal
2. `Current baseline` for what already exists
3. `Next work` for the remaining implementation
4. `Verification`
5. `Tests`
6. `Dependencies / Notes` when a package boundary or platform constraint matters

Treat the rewritten steps as a forward plan from the current codebase, not as a retrospective changelog.
